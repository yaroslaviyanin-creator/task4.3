В данном шаблоне проекта настроена сборка программы и библиотеки. Библиотека может собираться статической или динамической (опция BUILD_SHARED_LIBS).
Для программы и библиотеки настроена сборка тестов (опция BUILD_TESTING).

ГЕНЕРАЦИЯ ПРОЕКТА
Создает нужные файлы для сборки в отдельной папке build.

Выполнять из корневой директории проекта:
cmake -B build
(Опционально для конкретной версии студии: cmake -B build -G "Visual Studio 17 2022" -A x64)
(Опционально для динамической библиотеки: cmake -B build -DBUILD_SHARED_LIBS=ON)

Проект можно открыть в Visual Studio через файл CMakeLists.txt или сгенерированный .sln.

СБОРКА ПРОЕКТА
Компиляция исходного кода. Выполнять из корня проекта:
cmake --build build --config Debug
(Для релизной версии замените Debug на Release)

ЗАПУСК ТЕСТОВ
После успешной сборки можно запустить тесты через CTest. Выполнять из корневой директории:
cd build
ctest -C Debug

ЗАПУСК ПРОГРАММЫ И АРГУМЕНТЫ
Программа принимает параметры через аргументы командной строки:
Первый аргумент (обязательный) — имя текстового файла.
Второй аргумент (необязательный) — число. Если его указать, программа сама сгенерирует файл с таким количеством случайных слов.

Примеры использования (выполнять из корня проекта):

Пример А: У вас уже есть файл my_text.txt, и вы хотите его проанализировать
build\Debug\app.exe my_text.txt

Пример Б: Сгенерировать новый файл big_test.txt на 1 000 000 слов и распарсить его
build\Debug\app.exe big_test.txt 1000000
